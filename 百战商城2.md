### tabbar属性说明

在微信小程序开发中，`app.json` 是全局配置文件，而 `tabBar` 属性则是用来配置小程序底部（或顶部）的 Tab 栏，它决定了 Tab 栏的样式以及切换时显示的页面。

#### 属性详解

`tabBar` 属性是一个对象，包含以下常用配置项：

| 属性              | 类型     | 必填 | 默认值   | 描述                                          |
| ----------------- | -------- | ---- | -------- | --------------------------------------------- |
| `list`            | Array    | 是   | -        | Tab 列表，最少2个、最多5个。                  |
| `color`           | HexColor | 是   | -        | Tab 上文字的默认（未选中）颜色。              |
| `selectedColor`   | HexColor | 是   | -        | Tab 上文字选中时的颜色。                      |
| `backgroundColor` | HexColor | 是   | -        | Tab 的背景色。                                |
| `borderStyle`     | String   | 否   | `black`  | TabBar 上边框的颜色，仅支持 `black`/`white`。 |
| `position`        | String   | 否   | `bottom` | TabBar 的位置，仅支持 `bottom`/`top`。        |
| `custom`          | Boolean  | 否   | `false`  | 是否自定义 tabBar。                           |

#### `list` 数组项属性说明

`list` 是一个数组，数组中的每一项都是一个对象，代表一个 Tab 的配置。

| 属性               | 类型   | 必填 | 描述                                                  |
| ------------------ | ------ | ---- | ----------------------------------------------------- |
| `pagePath`         | String | 是   | 页面路径，必须在 `pages` 数组中先定义。               |
| `text`             | String | 是   | Tab 上显示的文字。                                    |
| `iconPath`         | String | 否   | 未选中时的图标路径。当 `position` 为 `top` 时不显示。 |
| `selectedIconPath` | String | 否   | 选中时的图标路径。当 `position` 为 `top` 时不显示。   |

#### 注意事项

- **数量限制**：`list` 数组中必须包含至少 2 个，最多 5 个 Tab。
- **图标规范**：`iconPath` 和 `selectedIconPath` 不支持网络图片，必须是本地文件路径。图片大小建议为 81px * 81px，且不超过 40kb。
- **顶部模式**：当 `position` 设置为 `top` 时，TabBar 将显示在顶部，并且不会显示图标，只显示文字。
- **自定义 TabBar**：如果将 `custom` 属性设置为 `true`，则表示使用自定义 TabBar。此时，你仍然需要声明 `color`、`list` 等基础配置项，但实际的渲染将由一个名为 `custom-tab-bar` 的自定义组件接管。

### swiper组件说明

在微信小程序开发中，`<swiper>` 组件是制作轮播图（Banner）、卡片滑动列表的核心容器。它只能放置 `<swiper-item>` 组件。

以下是 `<swiper>` 组件的核心属性详解及常用配置技巧：

#### 属性

| 属性名                     | 类型    | 默认值         | 说明                                 |
| -------------------------- | ------- | -------------- | ------------------------------------ |
| **indicator-dots**         | boolean | false          | 是否显示面板指示点（底部的小圆点）。 |
| **indicator-color**        | color   | rgba(0,0,0,.3) | 指示点颜色。                         |
| **indicator-active-color** | color   | #000000        | 当前选中的指示点颜色。               |
| **autoplay**               | boolean | false          | 是否自动切换。                       |
| **current**                | number  | 0              | 当前所在滑块的 index（从0开始）。    |
| **interval**               | number  | 5000           | 自动切换时间间隔（单位：毫秒）。     |
| **duration**               | number  | 500            | 滑动动画时长（单位：毫秒）。         |
| **circular**               | boolean | false          | 是否采用衔接滑动（无限循环）。       |
| **vertical**               | boolean | false          | 滑动方向是否为纵向。                 |
| **previous-margin**        | string  | "0px"          | 前边距，常用于露出前一项的一小部分。 |
| **next-margin**            | string  | "0px"          | 后边距，常用于露出后一项的一小部分。 |
| **display-multiple-items** | number  | 1              | 同时显示的滑块数量。                 |



#### 基础轮播图（带自动播放）

这是最常见的用法，通常用于首页 Banner。

**关键点**：设置 `autoplay` 为 `true`，并开启 `circular` 实现无缝循环。

```html
<swiper 
  class="swiper-container"
  indicator-dots="{{true}}" 
  autoplay="{{true}}" 
  interval="{{3000}}" 
  circular="{{true}}"
>
  <swiper-item wx:for="{{imgUrls}}" wx:key="*this">
    <image src="{{item}}" class="slide-image" />
  </swiper-item>
</swiper>
```

#### 卡片式滑动（露出边缘）

通过设置前后边距，可以让用户感觉到“后面还有内容”，常用于商品推荐或功能入口。

**关键点**：使用 `previous-margin` 和 `next-margin`。

```html
<swiper 
  class="card-swiper"
  previous-margin="30rpx" 
  next-margin="30rpx" 
  display-multiple-items="1"
>
  <swiper-item>
    <view class="card-item">内容 1</view>
  </swiper-item>
  <!-- 更多 item -->
</swiper>
```

#### 纵向滚动

用于新闻列表或垂直通告。

**关键点**：设置 `vertical="true"`。

```html
<swiper 
  class="vertical-swiper"
  vertical="{{true}}" 
  autoplay="{{true}}" 
  interval="{{2000}}"
>
  <!-- 列表项 -->
</swiper>
```

#### 常用事件说明

监听事件**bindchange**：即滑块切换时触发。

即滑块切换时触发。

- `event.detail = {current: 当前索引, source: 变化来源}`
- 常用于配合 `current` 属性实现联动（例如：点击底部导航切换轮播图）。

- **bindanimationfinish**: 动画结束时触发。
- **bindtransition**: 滑块位置发生改变时触发（高频触发，需注意性能）。

#### 注意事项

1. **子组件限制**：`<swiper>` 内部**只能**放置 `<swiper-item>` 组件，否则会导致未定义的行为。
2. **尺寸设置**：`<swiper>` 的高度默认是 150px（不同基础库可能略有差异），通常需要你在 CSS 中显式设置高度，否则图片可能显示不全。
3. **性能优化**：如果轮播图包含大量高清图片，建议开启 `skip-hidden-item-layout`（跳过未显示滑块的布局），这可以优化复杂情况下的滑动性能。

### 自定义组件及数据传递说明

微信小程序的自定义组件是将页面中可复用的部分（如弹窗、卡片、导航栏等）封装成独立模块的机制。它能有效提高代码复用性、增强可维护性，并使项目结构更清晰。

一个自定义组件通常由四个文件构成，与小程序页面的结构类似：

- **.json**: 组件配置文件。
- **.wxml**: 组件的模板结构。
- **.wxss**: 组件的样式文件。
- **.js**: 组件的逻辑脚本。

下面将以创建一个简单的卡片组件为例，详细介绍自定义组件的完整流程。

##### 创建自定义组件

1. **创建组件目录和文件** 在项目的 `components` 目录下（建议统一放在此目录），新建一个文件夹，例如 `my-card`，并在其中创建四个同名文件：

   - `my-card.js`
   - `my-card.json`
   - `my-card.wxml`
   - `my-card.wxss`

2. **编写组件文件**

   - **my-card.json (配置)** 这是声明组件最关键的一步，必须将 `component` 字段设置为 `true`。

     ```json
     {
       "component": true
     }
     ```

   - **my-card.wxml (模板)** 定义组件的 UI 结构。这里使用 `<slot>` 插槽，以便父组件可以传入自定义内容。

     ```html
     <view class="card">
       <view class="card-header">{{title}}</view>
       <view class="card-body">
         <slot></slot>
       </view>
     </view>
     ```

   - **my-card.wxss (样式)** 编写组件的专属样式。组件的样式默认是隔离的，只对自身生效。

     ```css
     .card {
       border: 1px solid #ddd;
       border-radius: 8px;
       padding: 20px;
       margin: 10px;
       background-color: #fff;
     }
     .card-header {
       font-size: 18px;
       font-weight: bold;
       margin-bottom: 10px;
       border-bottom: 1px solid #eee;
       padding-bottom: 10px;
     }
     ```

   - **my-card.js (逻辑)** 使用 `Component` 构造器定义组件，包括属性、数据和事件方法。

     ```javascript
     Component({
       // 1. 组件的属性，用于接收外部传入的数据
       properties: {
         title: {
           type: String,
           value: '默认标题' // 默认值
         }
       },
       // 2. 组件的内部私有数据
       data: {
         isExpanded: false
       },
       // 3. 组件的方法
       methods: {
         handleTap() {
           // 切换内部状态
           this.setData({
             isExpanded: !this.data.isExpanded
           });
           // 触发自定义事件，向父组件通信
           this.triggerEvent('myevent', { 
             expanded: this.data.isExpanded 
           });
         }
       }
     })
     ```

##### 使用自定义组件

1. **在页面中声明引用** 在需要使用该组件的页面（例如 `pages/index/index`）的 `.json` 配置文件中，通过 `usingComponents` 字段引入组件。

   ```json
   {
     "usingComponents": {
       "my-card": "/components/my-card/my-card"
     }
   }
   ```

   - `my-card` 是组件的标签名。
   - `/components/my-card/my-card` 是组件文件的相对路径。

2. **在页面模板中使用** 在页面的 `.wxml` 文件中，像使用内置组件一样使用自定义组件。

   ```html
   <!-- pages/index/index.wxml -->
   <view>
     <my-card title="我的卡片" bind:myevent="handleCardEvent">
       <view>这里是卡片的内容，通过插槽传入。</view>
       <button bindtap="handleTap">点击我</button>
     </my-card>
   </view>
   ```

   - `title="我的卡片"`：向组件的 `title` 属性传值。
   - `bind:myevent="handleCardEvent"`：监听组件内部触发的 `myevent` 事件。
   - `<view>...</view>`：这部分内容会通过 `<slot>` 插入到 `my-card` 组件内部。

3. **在页面逻辑中处理事件** 在页面的 `.js` 文件中，定义处理组件事件的方法。

   ```javascript
   // pages/index/index.js
   Page({
     handleCardEvent(e) {
       console.log('卡片状态变更:', e.detail.expanded);
     }
   })
   ```

##### 组件通信

组件化开发的核心是通信，主要有以下几种方式：

- **父传子 (属性传递)** 父组件通过在组件标签上绑定属性，将数据传递给子组件。子组件通过 `properties` 接收。这是最主要的数据流向。
- **子传父 (事件触发)** 子组件通过 `this.triggerEvent('事件名', 数据)` 方法触发自定义事件，将数据传递给父组件。父组件通过 `bind:事件名` 来监听并处理。
- **父调用子 (获取实例)** 父组件可以通过 `this.selectComponent('#id')` 或 `this.selectComponent('.class')` 获取子组件的实例，然后直接调用子组件的公共方法或修改其数据。

##### 组件生命周期

组件拥有自己的生命周期函数，可以在特定阶段执行相应的逻辑。这些函数定义在 `lifetimes` 字段中。

```javascript
Component({
  lifetimes: {
    created() {
      // 组件实例刚被创建，不能调用 setData
    },
    attached() {
      // 组件被插入到页面节点树中，常用于发起网络请求
    },
    ready() {
      // 组件在视图层布局完成，可以获取节点信息
    },
    detached() {
      // 组件被从页面节点树移除，用于清理工作
    }
  }
})
```
