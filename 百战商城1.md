### 微信网络请求API

##### wx.request简介

`wx.request` 是微信小程序中用于发起 HTTPS 网络请求的核心 API。通过它，小程序可以与后端服务器进行数据交互，实现数据的获取、提交等操作。

最基本的用法如下：

```javascript
# Get请求
wx.request({
  url: 'https://example.com/api/data', // 开发者服务器接口地址
  method: 'GET', // 请求方法，默认为 'GET'
  data: {
    // 请求的参数
    id: 123
  },
  success: (res) => {
    // 接口调用成功的回调函数
    if (res.statusCode === 200) {
      console.log('请求成功:', res.data);
    }
  },
  fail: (err) => {
    // 接口调用失败的回调函数
    console.error('请求失败:', err);
  },
  complete: () => {
    // 接口调用结束的回调函数（无论成功或失败都会执行）
    console.log('请求完成');
  }
});
# Post请求
wx.request({
  url: 'https://example.com/api/login',
  method: 'POST',
  data: {
    username: 'user1',
    password: '123456'
  },
  success: (res) => {
    console.log('登录结果:', res.data);
  }
});
# 请求头带token
const token = wx.getStorageSync('token'); // 从本地缓存获取 token
wx.request({
  url: 'https://example.com/api/userinfo',
  header: {
    'Authorization': `Bearer ${token}` // 将 token 放入请求头
  },
  success: (res) => {
    console.log('用户信息:', res.data);
  }
});
```

#####  wx.request配置参数

| 参数       | 类型          | 必填 | 默认值                               | 说明             |
| ---------- | ------------- | ---- | ------------------------------------ | ---------------- |
| `url`      | String        | 是   | -                                    | 请求地址         |
| `data`     | Object/String | 否   | -                                    | 请求参数         |
| `header`   | Object        | 否   | {'content-type': 'application/json'} | 请求头           |
| `method`   | String        | 否   | `GET`                                | 请求方法         |
| `timeout`  | Number        | 否   | `60000`                              | 超时时间(毫秒)   |
| `success`  | Function      | 否   | -                                    | 请求成功回调函数 |
| `fail`     | Function      | 否   | -                                    | 请求失败回调函数 |
| `complete` | Function      | 否   | -                                    | 请求结束回调函数 |

核心要点与最佳实践

1.  请求地址 ：**必须使用 HTTPS** 微信小程序强制要求所有网络请求都必须通过 HTTPS 协议进行，HTTP 请求将无法发送。 在开发阶段，可以在开发者工具中开启“不校验合法域名”选项进行调试。但在正式上线前，**必须**在微信公众平台后台的“开发设置”中配置服务器域名白名单，否则请求会失败。

2. 请求参数：GET 请求会自动拼接到 URL，POST 请求则放在请求体中。

3. 请求头：不能设置 `Referer`。常用于传递 Token。 Token称为令牌， 是服务器用于验证用户是否有访问权限的一串字符串密钥，对于需要用户登录的接口，通常需要在 `header` 中携带身份凭证（如 Token）。

4. 请求方法：支持 `GET`, `POST`, `PUT`, `DELETE` 等。

5. **请求成功返回的数据** 在 `success` 回调中，回调值 `res`  包含返回值 `data`、返回码`statusCode` 等，建议先检查 `res.statusCode` 是否为 `200`，再处理 `res.data`。这可以确保服务器从 HTTP 层面成功响应了你的请求。

6. **请求失败返回的数据** 在 `fail` 回调中，回调值 `err`包含 `errMsg`、`errno` 等。

   

##### wx.request封装示例

在实际项目中，为了代码的复用性和可维护性，通常会将 `wx.request` 进行封装，统一处理 loading 状态、错误提示和 Token 注入等逻辑。

以下是一个简单的 Promise 封装示例：

```javascript
// utils/request.js
function request(options) {
  // 显示加载提示
  wx.showLoading({ title: '加载中...' });

  return new Promise((resolve, reject) => {
    wx.request({
      url: options.url,
      method: options.method || 'GET',
      data: options.data || {},
      header: {
        'content-type': 'application/json',
        // 自动注入 token
        'Authorization': wx.getStorageSync('token') || ''
      },
      success: (res) => {
        if (res.statusCode === 200) {
          resolve(res.data); // 成功则 resolve 数据
        } else {
          wx.showToast({ title: '请求异常', icon: 'none' });
          reject(res); // 失败则 reject
        }
      },
      fail: (err) => {
        wx.showToast({ title: '网络错误', icon: 'none' });
        reject(err);
      },
      complete: () => {
        // 请求结束，隐藏 loading
        wx.hideLoading();
      }
    });
  });
}

export default request;
```

### ES6模块导入导出语法

ES6 模块（ES Module）是 JavaScript 官方的模块化标准，让代码可以拆分和复用，是现代前端开发（包括小程序、Vue、React）的基石。

ES6 模块的核心是围绕 `export`（导出）和 `import`（导入）这两个关键字展开的。

#### 导出 (export)语法

用于定义模块的对外接口，让其他文件可以使用本文件的功能

##### 导出方式1：命名导出 

 可以有**导出多个方法或者变量**

```javascript
// 写法1：逐一进行声明
export const PI = 3.14;
export function add(a, b) { ... }
export class Person { ... }
```

```javascript
// 写法2：末尾统一声明
const PI = 3.14;
function add(a, b) { ... }
class Person { ... }
// 统一导出
export { PI, add , Person };
```

##### 导出方式2：默认导出

特点： 一个模块只有一个默认导出。它通常用于导出模块最主要的功能。

```javascript
// main.js
export default function() {
  console.log('这是默认导出的函数');
}
// 或者导出一个值
// const config = { ... };
// export default config;
```

#### 导入 (import)语法

`import` 用于加载其他模块导出的功能。

**导入命名导出** 导入时必须使用**花括号 {}**，并且名字要与导出时一致。

```javascript
// 导入特定的命名导出
import { PI, add } from './utils.js';
// 导入时也可以重命名
import { PI as CircleRatio } from './utils.js';
```

**导入默认导出** 导入时**不使用花括号**，并且可以**任意命名**。

```javascript
// 导入默认导出，并自定义名称
import myFunc from './main.js';
```

**混合导入** 如果一个模块同时有默认导出和命名导出，可以这样导入：

```javascript
import myFunc, { PI, add } from './math.js';
```

**整体导入 (Namespace Import)** 将模块的所有**命名导出**作为一个对象导入。

```javascript
import * as utils from './utils.js';
console.log(utils.PI);
console.log(utils.add(1, 2));
```

#### 注意事项

`import` 和 `export` 语句必须位于模块的顶层，不能放在 `if` 语句或函数内部。



####  ES6 模块 vs CommonJS

ES6 模块与 CommonJS 规范的模块**语法区别**

| 特性         | ES6 模块                       | CommonJS                                        |
| ------------ | ------------------------------ | ----------------------------------------------- |
| **导出方式** | 使用 `export` / `import`。     | 使用 `module.exports` / `require()`。           |
| **主要优势** | 打包自动剔除未使用代码，体积小 | 语法简单直接                                    |
| **应用场景** | 大多数开发环境，主流语法       | Node.js 的传统标准，不支持ES6的环境（如浏览器） |
